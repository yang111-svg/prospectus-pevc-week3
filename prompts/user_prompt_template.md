# User Prompt Template

## 使用说明

将以下模板中的占位符替换为实际值后发送给LLM。

- {company_name} -- 公司全称
- {stock_code} -- 股票代码
- {text} -- 从招股说明书中提取的文本段落

---

## Prompt 模板

请从以下 {company_name}（股票代码：{stock_code}）的IPO招股说明书文本中，提取所有股本变化事件。

要求：
1. 识别所有增资、股权转让（出让方和受让方分别记录）、减资、设立出资等事件。
2. 股数统一转换为万股，金额统一转换为万元。
3. 每条记录必须包含原文证据片段（evidence_text）。
4. 输出为JSON数组格式。

公司名称：{company_name}
股票代码：{stock_code}

招股说明书文本如下：

{text}

请输出JSON数组：

---

## 变量说明

| 占位符 | 类型 | 示例 | 说明 |
|--------|------|------|------|
| {company_name} | string | "XX科技股份有限公司" | 公司全称，用于上下文锚定 |
| {stock_code} | string | "688XXX" | 股票代码，用于结果标识 |
| {text} | string | （招股书段落文本） | 待提取的招股说明书原始文本，通常为"股本演变"章节内容 |

## 注意事项

- {text} 的长度建议控制在 2,000-8,000 字符之间。若原文过长，建议按章节分段提取后合并结果。
- 每次调用应附带公司名称和股票代码，以便LLM在输出中正确填充对应字段。
- 若使用结构化输出（如 OpenAI function calling / JSON mode），可在模板末尾追加 schema 约束。
